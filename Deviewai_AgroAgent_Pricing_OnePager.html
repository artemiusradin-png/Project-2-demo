Assumptions & Exclusions

  • Assumes Salesforce exposes catalog / stock / farmland / soil data via usable REST APIs. If not, data-access work is +$15k – $30k (separate line, Phase 1).
  • Production LLM / vision API usage is billed to the client (usage-based; vision is token-heavy at 40k-user scale). Estimates indicative only.
  • Each phase is gated on a measurable acceptance metric. Phase 1: prescription accuracy ≥ 85% on a blind agronomist-panel test set. Phase 2: quote-build cycle time vs. baseline. Phase 3: data-mart freshness + tenant-isolation audit pass.
  • Phase-2 dependency: deterministic rules catalog (margin floors · pack sizes · regulatory compatibility) owned and sourced by the client before Phase 2 build start.
  • Phase-3 dependency: vendor data-sharing contracts in place before data-mart design.
  • Post-program maintenance & model-ops retainer: 15–25% of build / year — optional, separate SOW. Additional AI features (Blocks B, D) quoted incrementally.

▸ MARGIN PROFILE AT REV C IS THIN. Phase 1 margin ≈ $10–15k on $42k revenue; Phase 2 ≈ $5–9k; Phase 3 ≈ $3–5k. Total program margin ≈ $25k. Any scope creep eats the margin directly — hold scope hard and bill change requests separately.
▸ Hold per-phase floors: Phase 1 ≥ $38k · Phase 2 ≥ $18k · Phase 3 ≥ $13k. Below the floors the founder time + delivery risk stops being worth it.
▸ Sequencing lever: the phased structure IS the sales mechanism — Phase 1 proves accuracy, Phase 2 proves operational value before Phase 3 ROI is even discussed. Each gate is a re-close opportunity, not a risk.
▸ Reuse lever: Phases 2 & 3 ride Phase 1's substrate — no rebuild of connectors, vector index, auth, event log. ~70% (C) / ~80% (E) of code is reused. This is the architectural basis for the price drop.
▸ Discount lever: full-program commit-upfront pricing ($76k) trades $4k for a signed multi-phase commitment — useful when the client wants budget certainty. Hold $78k as the soft floor for the bundle.
